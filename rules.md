# Rules

- No inline styles.
- No redundant code.
- Always keep the code modular and componentized so that we do not end up with large files if possible.
- No duplicated code. Reuse components and repopulate them where possible.
- Keep the code clean and well commented.
- Keep the file structure clean and organized.
- Keep everything responsive across mobile, desktop, and tablet views.
- Always follow these rules and say `Rules followed` at the end of each summary.

# Deployment Note

- This project will be deployed on Railway.
- Do not hardcode secrets or database URLs in source files. Use environment variables instead.
